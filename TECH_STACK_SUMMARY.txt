================================================================================
                    VIBE-CODER TECHNOLOGY STACK SUMMARY
================================================================================

PROJECT TYPE: Multi-Agent Terminal Orchestration System
STATUS: Functional - Real terminals with file-based communication
READY FOR: AI agent integration

================================================================================
                              CORE ARCHITECTURE
================================================================================

FRAMEWORK:
  Primary: Node.js + Express (Backend)
  Frontend: React 19 + Vite
  Terminal: xterm.js (in browser via WebSocket)

TERMINAL IMPLEMENTATION:
  Technology: node-pty (native PTY binding)
  Real Shells: 5 actual /bin/bash sessions
  Connection: WebSocket bidirectional streaming
  Display: xterm.js with ANSI color support

API COMMUNICATION:
  Type: WebSocket (real-time) + REST (memory)
  WebSocket: Server/client terminal I/O streaming
  REST: /api/memory/* endpoints for agent state
  Message Format: JSON-based

INTER-AGENT COMMUNICATION:
  Type: File-based message broker
  Protocol: JSON files in shared-workspace/messages/
  Features: Direct messaging, broadcasting, inbox queuing

================================================================================
                         LANGUAGE & DEPENDENCIES
================================================================================

BACKEND DEPENDENCIES (3 core):
  - express@4.18        (HTTP server)
  - ws@8.16             (WebSocket library)
  - node-pty@1.0        (Terminal emulation)

FRONTEND DEPENDENCIES (2 core):
  - react@19.2          (UI framework)
  - react-dom@19.2      (React rendering)
  - xterm@5.3           (Terminal library, CDN)
  - vite@7.2            (Build tool, dev only)

AGENT STORAGE:
  - File system (JSON/MD)
  - ~200 JSON files per agent
  - No database required

================================================================================
                         DEPLOYMENT & HOSTING

CURRENT SETUP:
  Environment: Local development

  Frontend: Served from /public or Vite dev

CLOUDFLARE STATUS:

================================================================================
                         AGENT CONFIGURATION
================================================================================

AGENTS (5 total):
  1. Master - Orchestrates and coordinates
  2. Coder - Writes and implements code
  3. Researcher - Researches and documents
  4. Tester - Tests and ensures quality
  5. File Manager - Manages files and structure

AGENT ISOLATION:
  Config Dir: ~/.claude-agent-{name}
  Shells: Separate bash sessions per agent
  Environment: CLAUDE_CONFIG_DIR, SHARED_WORKSPACE, AGENT_NAME, etc.
  Authentication: Individual Claude Code CLI setup

AGENT COMMUNICATION:
  Type: File-based JSON messages
  Location: shared-workspace/messages/{agent}/
  Features: Direct, broadcast, inbox, task completion

================================================================================
                         MEMORY & PERSISTENCE
================================================================================

STORAGE STRUCTURE:
  Location: agent-memory/{agent}/
  Files Per Agent:
    - prompt-history.json (all prompts/responses)
    - memory-summary.md (human-readable overview)
    - context.json (current state)

MEMORY FEATURES:
  - Automatic command tracking
  - Full history (1000 prompt limit per agent)
  - Context preservation (CWD, files, sessions)
  - Search capability (keyword matching)
  - Export/import (JSON)

API ENDPOINTS:
  GET  /api/memory                    (all agents overview)
  GET  /api/memory/:agent             (specific agent memory)
  GET  /api/memory/:agent/history     (prompt history)
  GET  /api/memory/:agent/search?q=   (search)
  POST /api/memory/:agent/prompt      (save)
  DELETE /api/memory/:agent           (clear)

CLI TOOL: node memory-cli.js
